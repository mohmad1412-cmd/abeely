# إعداد حسابات الاختبار الحقيقية

## الأرقام المتاحة

تم إعداد رقمين حقيقيين للاختبار:

1. **0555111111** (966551111111)
   - الرمز: `0000`
   - الاسم: "مستخدم اختبار 1"
   - حساب حقيقي (ليس anonymous)

2. **0555222222** (966552222222)
   - الرمز: `0000`
   - الاسم: "مستخدم اختبار 2"
   - حساب حقيقي (ليس anonymous)

## كيفية الاستخدام

### 1. تسجيل الدخول

1. أدخل الرقم (مثلاً: `0555111111`)
2. أدخل الرمز: `0000`
3. سيتم تسجيل الدخول إلى حساب حقيقي

### 2. حفظ الاهتمامات

- الاهتمامات التي تحددها ستُحفظ في حسابك الحقيقي
- عند تسجيل الخروج والدخول مرة أخرى، ستجد اهتماماتك محفوظة

### 3. التحقق من الحساب

بعد تسجيل الدخول، افتح Console وابحث عن:
```
✅ TEST PHONE: verifyOtp successful - user logged in
isAnonymous: false  ← يجب أن يكون false
```

## الإعداد في Supabase

### Local Development

الرقمين موجودان في `supabase/config.toml`:
```toml
[auth.sms.test_otp]
966551111111 = "0000"
966552222222 = "0000"
```

### Production (Supabase Cloud)

1. اذهب إلى Supabase Dashboard
2. Authentication → Settings → SMS
3. أضف في Test OTP:
   ```
   966551111111 = 0000
   966552222222 = 0000
   ```
